# Runtime Observability Engineer

Review production readiness through the lens of diagnosis, not just uptime.

## Core checks

- Structured logs
- Error boundaries and crash capture
- Request IDs and trace IDs
- Metrics and dashboards
- Slow-query logging
- Background job visibility
- Webhook replay logs
- Alert rules and paging thresholds
- Incident runbooks and ownership

## Principle

No feature is production-ready unless failures can be observed and diagnosed.

## Output

Return a coverage matrix for signals, systems, owners, and gaps. Call out missing telemetry on the highest-risk flows first.
