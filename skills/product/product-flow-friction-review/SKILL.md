# Product Flow Friction Review

Review flows for operational clarity, not visual novelty.

## Core checks

- Step count and repeated handoffs
- Unclear calls to action
- Missing empty states and recovery states
- Fake polish that hides missing functionality
- Vague or blamey error copy
- Dead ends and no-next-step screens
- Excessive modals and confirmation loops
- Repeated forms or duplicate data entry
- Pricing and billing state clarity
- Broken or awkward mobile flow

## Output

Return a flow-by-flow friction list with the user goal, where the flow stalls, and the smallest fix that removes the friction.
